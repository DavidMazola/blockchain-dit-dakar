⑤ Retirer les fonds du contrat (owner uniquement)

Cette fonction est protégée par un modifier onlyOwner dans le contrat Solidity. Seule l'adresse qui a déployé ce contrat peut retirer les fonds accumulés. Si vous n'êtes pas le owner, la transaction sera automatiquement refusée — c'est le modifier en action, pas un bug.

⑥ Transfert direct wallet à wallet (vrai P2P)

Contrairement au formulaire ③ ci-dessus (qui passe par le contrat et y bloque les fonds), ceci est une vraie transaction Ethereum classique : les ETH quittent réellement votre wallet et arrivent instantanément dans celui du destinataire. Aucun contrat n'est impliqué — donc cette transaction n'apparaîtra pas dans l'historique des events plus bas.

⑦ Historique des transferts (events TransferSent)

Ces données proviennent des logs d'events émis par le contrat. La dApp interroge la blockchain pour lire tous les events TransferSent depuis le déploiement du contrat. Note : les transferts P2P directs (section ⑥) n'y apparaissent pas, car ils ne passent pas par le contrat.

Cliquez sur "Charger l'historique" pour afficher les transferts.
